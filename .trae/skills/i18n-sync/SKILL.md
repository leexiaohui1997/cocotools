# 多语言同步助手 (i18n-sync)

此技能用于在功能开发完成后，将简体中文（zh-CN）中新增的翻译键值同步到其他语言文件中，并自动进行翻译。

## 使用场景
- 当你在 `zh-CN.ts` 中添加了新的翻译内容后。
- 当你需要确保所有语言文件保持同步时。
- 当用户要求“同步语言包”或“翻译新增内容”时。

## 执行步骤

### 1. 确定基准文件
- 默认基准文件为 `apps/web/src/locales/zh-CN.ts`。
- 读取该文件内容，解析出所有的键值对。

### 2. 识别目标语言文件
- 查找 `apps/web/src/locales/` 目录下除 `zh-CN.ts` 以外的所有 `.ts` 文件（例如 `en-US.ts`, `ja-JP.ts`, `ar-KW.ts` 等）。

### 3. 对比与同步
- 逐个读取目标语言文件。
- 对比基准文件，找出目标文件中缺失的键（Key）。
- **注意**：保持文件结构的一致性（如嵌套对象结构）。

### 4. 翻译与写入
- 对于每一个缺失的键，根据其在 `zh-CN.ts` 中的中文值，将其翻译成目标语言。
- 将翻译后的键值对插入到目标文件的相应位置。
- 确保保留目标文件原有的内容，只添加新增的部分。

### 5. 验证
- 检查所有文件是否存在语法错误。
- 确保所有新增的键都已正确填充。

## 提示
- 在翻译技术术语时，请保持专业性。
- 对于 `ar-KW`（阿拉伯语）等从右向左（RTL）的语言，确保翻译符合语序习惯。
- 如果文件结构复杂，建议先读取文件内容，使用 AST 或正则匹配的方式进行插入，避免破坏文件格式。
